VI. Design Considerations

Objective


The primary objective of T3MPO was to develop a software interface that provides users the capability to control the playback of audio clips in Ableton, a popular digital audio workstation(DAW), in 3-dimensional space. To control clip playback, a 5 x 5 button grid was superimposed on a depth-map image of the user. The user can toggle the virtual buttons on or off in a similar fashion to their analog counterparts. That is, to toggle a virtual button, a user pushes it.


Constraints

Hardware Constraints:
Kinect Sensor: The Kinect is the 3D sensor used by T3MPO to retrieve and delineate the user's body position and gesture data.

The following computer hardware is required to use the Kinect:

  • Microsoft OS: Windows XP, Windows 7
  • Microsoft OS: OS X 10.6 or later
  • 32 bit (x86) or 64 bit (x64) processor
  • Dual-core 2.66-GHz or faster processor
  • Dedicated USB 2.0 bus
  • 2 GB RAM

A number of software libraries and products from 3rd parties are required for developing and using T3MPO.

OpenNI: Open source software development kit for 3D Sensors.
Synapse: A software module built using OpenNI framework and has message sending capabilities for communication with DAWs.

Design Constraints:

The primary design constraint of T3MPO was to program a 5x5 button grid for the T3MPO user interface. This button grid is superimposed upon the depth map of the user and provides visual feedback with regards to what audio clips are toggled on or off. Each button in the first four columns, from left to right, represents an audio clip that may be toggled on or off. When an audio clip is off, the respective button is grey. Next, when a user toggles a button on, the button color changes to yellow and vise-versa. The final column of buttons is used to toggle all the clips on or off in a respective row. We refer to rows as scenes. When a scene is launched, or turned on, the corresponding scene control button changes color from gray(the off-state) to magenta (the on-state). To allow users to stop all audio playback at once, a global stop button was added to the upper-left corner of the user interface. This stop button is red. Initially, its opacity is about 50%. When the stop button is activated by the user, its opactiy briefly goes to 100%, thus providing the user with feedback that the stop command has been acknowledged.
